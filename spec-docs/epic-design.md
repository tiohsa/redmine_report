# Epic: DESIGN.md準拠で redmine_report のデザイン基盤・レイアウト・タイポグラフィを再設計する

## 背景
redmine_report のUIは、画面ごとの個別調整が増えやすく、デザイン・余白・文字階層・コンポーネント表現の統一が不十分になりやすい。

今回、DESIGN.md を基準デザインとして採用し、以下の観点で redmine_report の見た目と実装基盤を再設計する。

- 白主体のクリーンな画面構成
- 青系アクセントと近黒テキストによる情報の明瞭化
- pill型UIや大きめの角丸による一貫したコンポーネント表現
- タイポグラフィの役割分担
- 広い余白と階層的な情報配置

ただし、DESIGN.md はプロダクトショーケース寄りの世界観を含むため、それをそのまま模倣するのではなく、redmine_report のような業務レポートUIに適した情報密度・可読性・保守性を優先して適用する。

## 目的
DESIGN.md の設計思想をもとに、redmine_report のデザイン基盤を整理し、以下を実現する。

1. 画面全体で色・余白・角丸・影・文字階層を統一する
2. 共通コンポーネント中心のスタイル設計へ移行する
3. レポート画面で「結論 → KPI → 課題 → 根拠」の順に読みやすい構造へ改善する
4. 将来の画面追加や改善でもデザインの一貫性を維持できる状態にする

## スコープ
### 対象
- デザイントークン定義
- タイポグラフィ設計
- 共通コンポーネントの見直し
- 主要レポート画面のレイアウト改善
- KPI/サマリー/表/モーダルなど主要UIの統一
- レスポンシブと表示崩れ確認
- 主要画面の見た目回帰確認

### 非対象
- レポートロジックや集計ロジックの仕様変更
- 新しい分析機能の追加
- グラフライブラリ自体の全面置換
- ダークテーマ対応の新規実装（必要なら別Epic）

## デザイン適用方針
### 採用するもの
- 白主体の背景構成
- 青系ブランドアクセント
- 近黒の見出し/本文色
- pill型ナビ/フィルタUI
- 8px基準の余白スケール
- 大きめの角丸カード
- 見出し・本文・データ表示の役割分担

### 抑制するもの
- 彩度の高い装飾表現の多用
- 業務画面で過剰なグラデーション演出
- 多フォントの過剰適用
- 情報密度を下げる過剰な余白

## タイポグラフィ方針
- 本文/UI: DM Sans + Noto Sans JP を基本とする
- ページタイトル/主要見出し: Outfit を限定利用する
- データ/表/技術情報: Roboto を必要箇所で利用する
- Poppins は限定利用とし、多用しない
- 日本語の本文や表は可読性優先で 16px を基本とする

## 実装方針
1. 既存UIとスタイル資産を棚卸しする
2. DESIGN.md を元にデザイントークンを定義する
3. タイポグラフィルールを定義する
4. 共通コンポーネントを再設計する
5. トップレポート画面から段階適用する
6. KPI/表/モーダルなどへ順次展開する
7. レスポンシブ・崩れ・回帰確認を実施する

## リスク
- 情報密度が下がり、業務レポートとして見づらくなる
- フォント導入が増え、表示性能が悪化する
- ページ固有CSSとの競合で崩れが発生する
- 見た目重視になり、結論の見やすさが落ちる

## リスク対策
- 情報優先順位を先に定義する
- フォントは最小構成で導入する
- 共通トークン → 共通部品 → 画面適用の順を守る
- 主要画面で回帰確認を行う

- デザイン変更がコンポーネント/トークン単位で保守可能になっている
- エグゼクティブ向けに見やすいサマリー導線ができている
